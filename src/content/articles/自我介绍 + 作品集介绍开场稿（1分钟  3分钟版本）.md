# 一、1 分钟版本

大家好，我叫 [你的名字]，目前在准备系统策划方向的岗位。  
我这次的作品集主要围绕三款不同类型的长线产品做了系统拆解，分别是《崩坏：星穹铁道》《明日方舟：终末地》和《王者荣耀国际服》。

我做这套作品集的目的，不是单纯写游戏评测，而是想用系统策划的视角去分析不同产品是怎么把单次游玩体验转化成长期留存的。  
所以我在拆解时会重点关注几个问题：

- 产品的核心体验是什么
- 玩家主路径是什么
- 哪个系统真正承担了中层驱动
- 资源、成长和目标之间是否形成了闭环
- 当前系统最大的风险点是什么，以及可以怎么优化

这三款游戏分别对应了三类产品结构：
- 《崩坏：星穹铁道》偏养成驱动
- 《明日方舟：终末地》偏多系统联动
- 《王者荣耀国际服》偏竞技外围和赛季社交驱动

我希望通过这套作品集体现自己在系统拆解、玩家行为分析、留存逻辑判断和优化思考方面的能力。  
如果您愿意的话，我可以先从其中一款开始详细展开。

# 三、3 分钟版本

面试官您好，我叫 [你的名字]，目前主要在准备系统策划方向的岗位。  
我这次的作品集，主要是围绕三款不同类型的长线产品做了系统拆解，分别是《崩坏：星穹铁道》《明日方舟：终末地》和《王者荣耀国际服》。

我之所以做这套作品集，是因为我希望自己在准备系统策划岗位时，不只是停留在“我玩过很多游戏”或者“我对某款游戏很熟”这种层面，而是能够用比较统一的方法去分析不同产品的系统结构。  
所以我在做这三份拆解时，会刻意使用同一套分析主线：

先看产品的核心体验是什么，  
再看玩家主路径是怎样形成的，  
然后判断哪一套系统真正承担了中层驱动，  
接着分析资源、成长、留存和目标结构，  
最后再去识别问题和提出优化建议。

我选择这三款游戏，也是有明确分工的。

第一款是《崩坏：星穹铁道》。  
我把它当作长线养成产品的代表来拆，重点分析了它的回合制战斗、角色养成、体力资源、版本角色和商业化联动。我想通过这篇体现自己对养成系统、资源节奏和版本驱动逻辑的理解。

第二款是《明日方舟：终末地》。  
这篇我更关注它作为复合系统型产品的结构潜力，也就是战斗、探索、采集、基地和制造系统之间能不能形成真正闭环。我想用这篇体现自己对多系统联动、复杂度控制和结构推演的理解。

第三款是《王者荣耀国际服》。  
这篇我主要不是分析局内英雄平衡，而是分析竞技产品的外围系统，比如新手承接、英雄池、排位赛季、活动、社交和身份表达。我想通过这篇说明，我也能理解竞技服务型产品是怎么把高频对局转化成长期留存的。

我希望这套作品集最终展示的是三类能力：

第一，结构化拆解能力。  
也就是我能把一个复杂产品拆成核心玩法、系统结构、玩家路径、资源闭环和长期目标。

第二，问题诊断能力。  
我不会只停留在“这个地方不舒服”，而是尽量把体验问题转化成系统问题，比如成长反馈不足、复杂度释放不合理、目标断层或者挫败缺少缓冲。

第三，优化思考能力。  
也就是在提出建议时，不只是说“加奖励”或者“优化体验”，而是会尽量去想这个调整会影响哪些玩家行为、会不会破坏原有节奏、有没有副作用。

整体来说，我做这套作品集，是想证明自己不只是会写分析文章，而是已经在用系统策划的视角去理解产品。  
如果您愿意的话，我可以接下来先讲《崩坏：星穹铁道》，因为那篇最适合完整展开我的分析框架；如果您更想看多系统联动或者竞技产品的思路，我也可以直接讲《终末地》或者《王者荣耀国际服》。

# 四、3 分钟版本（更偏“介绍作品集”而非“纯自我介绍”）

我这次的作品集，核心目标其实很明确，就是想用三款不同类型的游戏，来展示我对系统策划岗位的理解。

我选的三款分别是《崩坏：星穹铁道》《明日方舟：终末地》和《王者荣耀国际服》。  
之所以这样选，是因为这三款游戏刚好代表了三种不同的产品结构：

- 《崩坏：星穹铁道》代表长线养成驱动型产品  
- 《明日方舟：终末地》代表复合系统联动型产品  
- 《王者荣耀国际服》代表竞技服务型产品

我希望通过这三款游戏说明，我不是只会分析某一个赛道，而是能用统一的系统策划框架，去理解不同产品是怎么组织玩家行为的。

我在拆解时基本都会围绕同一条主线去做：

先看产品的核心体验，  
再看玩家为什么会持续重复游玩，  
然后找出真正承担中层驱动的系统，  
再去分析资源、成长、留存和长期目标结构，  
最后判断主要问题和优化方向。

比如在《崩坏：星穹铁道》里，我重点看的是它如何通过角色养成和体力系统，把战斗、资源、版本角色和商业化稳定串成闭环。  
在《明日方舟：终末地》里，我更关注的是它能不能让战斗、探索、采集、基地和制造真正构成长期循环，而不是一堆并列系统。  
在《王者荣耀国际服》里，我则更强调对局之外的赛季、活动、社交和身份表达，因为我觉得竞技产品长期能不能留住人，关键不只是对局本身，而是外围系统能不能承接竞技压力。

我希望面试官看到这套作品集之后，能感受到三点：

第一，我有比较清晰的系统拆解框架。  
第二，我能把玩家体验问题转化成系统结构问题。  
第三，我能在提出优化建议时考虑节奏、联动和副作用，而不是只停留在表面意见。

如果需要的话，我可以先从这三款里任选一款展开讲，或者我也可以先讲一下我做这套作品集后，对系统策划这个岗位的理解。

# 五、如果面试官追问“你为什么想做系统策划”，可以接这段

## 补充话术

我之所以更想做系统策划，是因为我比较喜欢从“结构”角度看问题。  
相比单纯去讨论某个功能好不好，我更关注的是：

- 这个系统为什么存在
- 它驱动了玩家什么行为
- 它和其他系统怎么联动
- 它为什么会产生现在这种体验
- 如果要改，应该改目标、规则、资源还是节奏

我在做这套作品集的过程中，也越来越确认自己对这种分析方式是有兴趣的。  
尤其是在拆《终末地》和《王者荣耀国际服》的时候，我会明显感受到，不同产品虽然玩法差异很大，但系统策划真正关心的问题其实是相通的，这也让我更想往这个方向继续发展。

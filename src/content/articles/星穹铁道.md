# 《崩坏：星穹铁道》系统策划向完整拆解报告

> 说明：
> - 本报告从系统策划岗位视角出发，重点分析《崩坏：星穹铁道》的核心玩法循环、养成系统、资源结构、长期目标、商业化协同与可优化空间。
> - 报告目标不是评价“好不好玩”，而是拆解“系统为什么这样设计、如何驱动玩家行为、存在什么问题、可以如何优化”。
> - 文中部分内容基于产品长期运营阶段的通用系统观察与策划视角推演，适合作为面试作品底稿。

# 一、项目基础信息

## 1.1 基础信息

| 项目 | 内容 |
|---|---|
| 游戏名称 | 崩坏：星穹铁道 |
| 类型 | 回合制队伍 RPG / 服务型二次元长线运营产品 |
| 平台 | 移动端 / PC / 主机（多端） |
| 核心商业模式 | 免费下载 + 角色/光锥抽卡 + 月卡 + 通行证 + 限时礼包 |
| 核心体验 | 回合制战斗、角色收集、养成成长、版本叙事、阶段性挑战 |
| 目标用户 | 喜欢二次元角色、养成收集、轻中度日常、剧情体验与队伍构筑的玩家 |
| 竞品参考 | 原神、FGO、阴阳师、崩坏3、明日方舟、其他服务型二游 |

## 1.2 一句话总结

《崩坏：星穹铁道》是一款以“低操作门槛的回合制战斗 + 角色养成收集”为基础，通过“体力驱动资源获取、遗器随机构筑、版本更新与高难玩法”维持长期留存，并通过“角色投放与叙事塑造”提升商业化转化的服务型 RPG 产品。

## 1.3 选择该游戏作为拆解对象的原因

- 系统结构完整，包含战斗、养成、资源、任务、活动、商业化、长线挑战等典型模块
- 作为成熟长线运营二游，系统目标明确，适合做系统策划分析
- 其“低门槛回合制 + 中深度养成 + 高频版本更新”的结构具有较强代表性
- 存在明显的系统优点与典型问题，例如遗器随机性、体力限制、养成压力、长草期等，适合提出优化方案

# 三、目标用户与产品定位

## 3.1 目标用户画像

### 3.1.1 核心用户特征

- 喜欢二次元角色、美术包装、世界观叙事
- 能接受长线养成与资源积累
- 喜欢“轻操作、重构筑”的战斗体验
- 日常在线时间中等，愿意形成固定上线习惯
- 对角色收集、强度构筑、版本内容有持续关注

### 3.1.2 次核心用户

- 对米哈游品牌或角色塑造有偏好的泛二游用户
- 偏剧情党、角色党、轻度养成玩家
- 对回合制门槛低、适合碎片化游玩有需求的用户
- 曾接触开放世界或动作类产品，但希望负担更低的玩家

### 3.1.3 用户需求拆分

| 需求类型 | 表现 |
|---|---|
| 收集需求 | 抽取角色、光锥、遗器追求毕业 |
| 成长需求 | 角色提升、队伍完整、伤害增长、通关更高难内容 |
| 叙事需求 | 主线剧情、角色塑造、版本事件 |
| 轻量陪伴需求 | 每日上线有明确目标，日常结构固定 |
| 构筑需求 | 配队、属性克制、战技点分配、遗器词条选择 |
| 挑战需求 | 混沌回忆、虚构叙事、末日幻影、模拟宇宙等 |

## 3.2 产品定位判断

### 3.2.1 产品核心定位

《崩坏：星穹铁道》并不是强调高操作深度的战斗产品，而是强调：

- 低操作门槛
- 高角色价值
- 中长期成长积累
- 高频版本内容供给
- 角色投放与剧情包装协同

### 3.2.2 其核心竞争力

- 高规格美术与演出包装降低内容消费门槛
- 回合制天然适配移动端和轻操作用户
- 角色设计、叙事、数值成长和商业化协同度高
- 日常结构清晰，形成稳定上线习惯
- 长期玩法通过多种挑战模式维持中后期目标

# 四、整体系统结构总览

## 4.1 主要系统构成

从系统策划角度看，本作核心系统可划分为以下八类：

1. 核心战斗系统
2. 角色养成系统
3. 资源与体力系统
4. 剧情与地图推进系统
5. 任务与日常系统
6. 长期挑战系统
7. 活动系统
8. 商业化系统

## 4.2 系统关系总览

玩家进入游戏  
→ 推进剧情、探索地图、参与战斗  
→ 获取角色、资源、材料、货币  
→ 投入角色、光锥、遗器、行迹等养成  
→ 提升队伍强度与配队完整性  
→ 挑战更高难度内容或更复杂机制副本  
→ 获取更高价值奖励  
→ 通过体力、日常、周常、活动维持回流  
→ 通过版本更新、新角色、新剧情、新玩法延长生命周期

## 4.3 系统设计目标判断

| 系统      | 主要作用      | 对玩家的意义      | 对产品的意义      |
| ------- | --------- | ----------- | ----------- |
| 战斗系统    | 提供基础玩法体验  | 体验角色与队伍构筑价值 | 构成主游玩场景     |
| 养成系统    | 提供中长期成长目标 | 角色变强、配队完善   | 延长生命周期、创造消耗 |
| 体力与资源系统 | 控制成长速度    | 形成每日目标与资源取舍 | 稳定留存节奏      |
| 长期挑战系统  | 提供高端目标    | 验证养成成果、追求毕业 | 稳定中后期活跃     |
| 活动系统    | 提供阶段性新鲜感  | 获得奖励、体验新内容  | 缓解长草，拉动回流   |
| 商业化系统   | 提供付费入口    | 获取喜欢角色、加速成长 | 支撑营收与版本投放   |

# 五、核心玩法循环拆解

# 5.1 分钟级循环

单次战斗或副本的基础循环为：

进入战斗  
→ 观察敌方弱点、机制与行动顺序  
→ 选择普攻、战技、终结技、秘技等操作  
→ 管理战技点和大招释放时机  
→ 击破弱点、控制敌方节奏、承伤与回复  
→ 完成战斗结算  
→ 获取资源或推进内容

## 5.1.1 分钟级体验的核心乐趣

- 角色技能演出反馈强
- 敌我行动顺序与弱点击破形成策略层
- 队伍协同构筑带来“战前决策 + 战中验证”的满足感
- 战斗规则简单，降低操作门槛
- 终结技插入机制提升关键时刻决策感

## 5.1.2 设计评价

优点：
- 门槛低，容易让大多数用户理解并参与
- 短战斗中也有一定策略深度，不完全依赖数值碾压
- 角色差异与技能机制可被明确感知，有利于角色价值塑造

问题：
- 当玩家数值形成压制后，低难内容战斗决策深度下降
- 部分资源副本重复频率高，分钟级体验新鲜度不足
- 自动战斗功能降低操作负担的同时，也削弱了一部分系统参与感

# 5.2 日循环

玩家每日常见路径如下：

登录  
→ 领取上线奖励或处理消息  
→ 消耗开拓力刷取养成材料  
→ 完成每日实训、任务与活跃目标  
→ 处理模拟宇宙/周本/限时活动  
→ 强化角色、光锥、遗器、行迹  
→ 检查高难玩法进度或版本目标  
→ 退出并等待下一轮资源刷新

## 5.2.1 日循环的设计目标

- 为玩家提供清晰稳定的上线理由
- 控制日均时长，避免过度肝度
- 通过开拓力限制资源获取速度
- 让玩家持续积累、逐步接近养成目标

## 5.2.2 日循环评价

优点：
- 结构稳定、理解成本低
- 任务目标明确，适合形成习惯
- 日常负担相较传统重肝产品更可控

问题：
- 中后期日常行为同质化明显
- 资源刷取与自动战斗绑定后，日常更偏“执行型”
- 若版本活动空档期较长，玩家目标感会明显下降

# 5.3 周循环

每周循环通常包括：

- 模拟宇宙积分与周常奖励
- 周本材料获取
- 高难玩法刷新或阶段性挑战
- 通行证周任务推进
- 活动阶段目标

## 5.3.1 周循环的价值

- 把日常短目标提升到中周期目标
- 让玩家在一周内形成资源规划和养成安排
- 为中后期用户提供“本周要完成什么”的明确框架

# 5.4 版本循环

典型版本循环为：

版本前瞻  
→ 新角色曝光  
→ 新剧情 / 新地图 / 新活动上线  
→ 玩家抽取新角色  
→ 刷取新角色养成材料  
→ 用新角色参与活动与挑战玩法  
→ 版本后期进入长草阶段  
→ 等待下个版本更新

## 5.4.1 版本循环的设计本质

这款产品的长期留存本质上高度依赖版本供给。  
版本的职责不仅是“加内容”，更是：

- 刷新角色需求
- 刷新配队环境
- 刷新资源需求
- 刷新叙事关注度
- 刷新抽卡动机
- 刷新老玩家回流理由

## 5.4.2 核心循环总结

《崩坏：星穹铁道》通过“低门槛回合制战斗”提供短期反馈，通过“角色多线养成 + 体力限制 + 长期挑战”建立中期成长目标，通过“高频版本更新 + 新角色投放 + 叙事驱动”维持长期留存。  
其循环优势在于结构清晰、角色价值高、运营协同强；主要风险在于中后期重复刷取感重、版本空档时内容消耗快、养成随机性带来的挫败感累积。

# 六、核心战斗系统拆解

# 6.1 战斗系统基本结构

## 6.1.1 核心规则

- 回合制战斗
- 队伍通常由 4 名角色组成
- 角色定位包括主 C、副 C、辅助、治疗、护盾、控制等
- 战斗中核心资源包括战技点、终结技能量、行动顺序、弱点击破
- 属性克制与弱点系统影响战斗效率和策略选择

## 6.1.2 核心机制

- 普攻：基础行动，通常回复战技点
- 战技：消耗战技点，体现角色核心能力
- 终结技：高价值爆发或功能技能
- 天赋：塑造角色独特机制
- 秘技：战前优势或功能性切入
- 弱点击破：提高属性与配队策略价值
- 行动顺序：影响节奏控制与爆发窗口

# 6.2 系统目标

从系统策划视角，战斗系统承担的核心职责是：

- 提供低门槛、可视化的基础乐趣
- 让角色机制差异被明确感知
- 为养成系统提供验证场景
- 为新角色销售提供体验承载
- 为长期玩法提供机制深度

# 6.3 战斗系统优点

## 6.3.1 易上手

相比动作游戏，回合制大幅降低了输入门槛。  
玩家只要理解基础按钮、属性、技能功能，就能快速进入有效战斗。

## 6.3.2 构筑感强于操作压力

这让系统策划可以把重心放在：

- 角色差异
- 配队逻辑
- 技能联动
- 资源管理
- 敌方机制设计

而不必过度依赖操作精度。

## 6.3.3 角色价值表达清晰

每个角色都能通过战技、终结技、天赋在战斗中形成记忆点，这对角色售卖、版本推广和用户情感建立非常重要。

# 6.4 战斗系统问题

## 6.4.1 低难内容决策退化

在日常副本、重复刷本场景中，玩家很快会形成固定流程，策略空间下降。  
自动战斗进一步放大了这一问题。

## 6.4.2 角色设计强依赖数值与机制包装

由于战斗操作空间有限，新角色的吸引力需要更依赖：
- 数值效率
- 机制独特性
- 动画演出
- 环境适配度

这会加大版本平衡与角色投放压力。

## 6.4.3 高难度机制表达容易依赖环境压迫

当需要制造挑战时，设计可能倾向于：
- 提高数值门槛
- 强化环境限制
- 通过机制迫使特定角色上场

若处理不当，容易让玩家感到“不是在做策略，而是在做角色检测”。

# 6.5 战斗系统优化建议

- 在资源副本中加入更轻量的可变机制，提高重复刷取中的微策略差异
- 扩展部分角色的通用价值，减少环境过窄导致的角色淘汰感
- 为自动战斗场景提供更多“自动策略预设”，增强玩家配置感而非纯放置感

# 七、养成系统拆解

# 7.1 养成系统组成

《崩坏：星穹铁道》的养成系统是其长线结构核心，主要包括：

- 角色等级
- 角色突破
- 行迹系统
- 光锥等级与突破
- 遗器系统
- 队伍搭配与角色补位
- 命途/星魂等深化能力模块

# 7.2 系统设计目标

养成系统的目标不只是“让玩家变强”，而是同时完成以下任务：

- 提供长期资源消耗口
- 构建稳定日常刷取理由
- 拉长新角色从“获取”到“可用”再到“毕业”的路径
- 让玩家在角色喜好和资源效率之间做决策
- 为商业化留出持续转化空间

# 7.3 养成路径分析

典型角色养成路径：

抽到角色  
→ 升级角色等级  
→ 完成突破  
→ 提升行迹与关键技能  
→ 配置可用光锥  
→ 配置主属性正确的遗器  
→ 优化遗器副词条  
→ 根据高难玩法继续精细化调优

## 7.3.1 路径特点

这是一个典型的“多层递进型养成结构”：

- 前期：快速建立战力
- 中期：材料卡点明显
- 后期：遗器和词条毕业成为主要追求

# 7.4 养成系统优点

## 7.4.1 多层目标清晰

角色的每个养成层都有明确作用，玩家可以理解“我为什么要刷这个材料”。

## 7.4.2 阶段性目标丰富

即使无法立刻毕业，玩家也能感受到：
- 先能上场
- 再补技能
- 再补装备
- 再追求毕业词条

这种多阶段推进很适合长期运营。

## 7.4.3 强化角色获取后的持续价值

新角色不是抽到即结束，而是进入一段完整养成周期。  
这增加了内容消耗深度，也延长了新角色带来的活跃期。

# 7.5 养成系统问题

## 7.5.1 养成线较多，理解成本中等偏高

对新玩家而言，角色等级、突破、行迹、光锥、遗器、套装、速度阈值、词条收益等信息需要逐步理解，学习成本不低。

## 7.5.2 新角色“抽到了但养不起”

这是该产品很典型的问题：
- 抽取新角色会带来强烈即时满足
- 但角色真正进入可用状态需要大量资源
- 若玩家库存不足，会产生“抽到后短时间内无法上场”的落差

## 7.5.3 遗器系统随机性强

遗器随机性承担了长线留存的重要功能，但也带来：
- 重复刷取疲劳
- 成本投入与结果脱钩
- 副词条成长不可控
- 角色毕业周期不确定

## 7.5.4 多队伍环境放大资源压力

当高难玩法要求多支队伍时，单角色养成问题会进一步扩展为：
- 多角色同时养成
- 多个光锥分配
- 遗器共用与抢占
- 功能角色投入不足

# 7.6 养成系统结论

《崩坏：星穹铁道》的养成系统是典型的“前中后期都能持续消耗资源”的成熟长线设计。  
它的优点在于目标层次清晰、角色价值稳定、成长过程可切片；  
问题在于新角色启用成本高、随机毕业压力大、多队需求对资源池消耗过快。

| 问题 | 影响 | 优化建议 | 预期结果 |
|---|---|---|---|
| 新角色养成周期长 | 抽到后短期无法体验完整强度 | 增加版本角色定向成长支持包或试用转化养成任务 | 缩短获取到可用的断层 |
| 遗器随机性过强 | 重复刷取疲劳、挫败感上升 | 增强定向词条保底或阶段性纠偏机制 | 保留长线追求，同时减少极端非酋体验 |
| 多队培养压力高 | 中后期资源吃紧，挤压尝试空间 | 提升功能角色最低可用门槛的资源友好度 | 鼓励更多配队尝试 |

# 八、资源与经济系统拆解

# 8.1 资源系统构成

主要资源可分为以下几类：

| 资源类型 | 主要用途 |
|---|---|
| 开拓力 | 参与材料副本、遗器本等日常资源产出 |
| 信用点 | 通用养成货币 |
| 经验材料 | 角色升级 |
| 光锥经验材料 | 光锥升级 |
| 突破材料 | 角色/光锥突破 |
| 行迹材料 | 技能与天赋强化 |
| 遗器 | 装备毕业与后期构筑 |
| 星琼/票券 | 抽卡资源 |
| 周本材料 | 高阶技能成长 |

# 8.2 体力系统分析

开拓力是本作资源节奏控制的核心。

## 8.2.1 体力系统目标

- 限制单日成长速度
- 提供稳定每日上线动机
- 迫使玩家做资源规划
- 避免内容被短期消耗殆尽
- 与版本投放节奏形成协同

## 8.2.2 体力系统效果

正面效果：
- 降低整体肝度
- 强化日常节奏感
- 让玩家更重视资源分配

负面效果：
- 当玩家有明确养成目标时，体力不足带来的阻塞感明显
- 新角色版本期间，资源需求集中爆发
- 长期来看，日常容易变成“上线清体力”的流程化行为

# 8.3 资源闭环分析

剧情推进、日常副本、活动、周常挑战提供资源  
→ 资源投入到角色、光锥、行迹、遗器养成  
→ 队伍强度提升  
→ 可挑战更高难度内容  
→ 获得更高价值奖励或更完整版本体验  
→ 继续投入新角色或补强旧角色

# 8.4 资源卡点设计

## 8.4.1 典型卡点

- 行迹材料
- 遗器毕业效率
- 信用点大量消耗
- 新角色与老角色资源冲突
- 多队构筑下材料分配不足

## 8.4.2 卡点设计的合理性

合理的一面：
- 使玩家始终有中期目标
- 避免角色过快毕业
- 强化版本更新期间的持续活跃

不合理的一面：
- 若卡点过于集中，会使“抽到角色”转化为“新增负债”
- 当玩家对结果不可控时，卡点更容易转化为疲劳而非期待

# 8.5 经济健康度判断

总体来看，《崩坏：星穹铁道》的经济系统是健康的，主要表现为：

- 资源消耗口明确且长期稳定
- 体力系统能有效控制产出节奏
- 大多数资源都存在明确去向，不易废弃

但也有问题：
- 遗器词条随机导致部分产出价值低
- 新角色版本期间的集中消耗会放大紧缺感
- 养成资源结构对轻度玩家尚可，对追求多角色并行养成玩家压力较大

# 8.6 资源系统优化建议

- 增加“新角色上线期”的限时定向养成支持，降低版本初期断层
- 为遗器系统增加更透明的阶段性保底反馈，提升投入可感知性
- 通过活动更精准投放中后期紧缺资源，而非泛化发放低价值材料

# 九、剧情推进与地图探索系统

# 9.1 系统定位

虽然《崩坏：星穹铁道》不是开放世界产品，但其剧情与地图探索承担了非常重要的功能：

- 构建世界观与角色情感链接
- 提供版本内容消费主轴
- 支撑活动、任务、角色投放的包装层
- 在战斗与养成之外提供节奏变化

# 9.2 设计价值

剧情与地图系统在本作中不是独立存在，而是与商业化、留存、角色塑造高度耦合。

具体表现为：
- 新角色上线往往伴随剧情强化认知
- 地图与任务结构帮助玩家理解版本主题
- 叙事增加角色抽取的情感动机
- 主线推进本身就是回流理由

# 9.3 问题点

- 对部分非剧情型玩家来说，主线推进节奏可能与养成节奏不完全一致
- 当剧情消费结束后，地图探索的重复价值有限
- 叙事强度依赖版本质量，波动会直接影响玩家对版本满意度

# 十、任务系统与日常结构

# 10.1 任务系统构成

主要包括：

- 主线任务
- 冒险任务 / 同行任务
- 每日实训
- 周常目标
- 通行证任务
- 活动任务

# 10.2 系统目标

任务系统的核心职责是：

- 引导玩家行为
- 规范每日上线路径
- 补充资源投放
- 提供阶段性目标与完成感

# 10.3 设计优点

- 日常任务简洁，门槛低
- 与开拓力、活跃、资源获取联动紧密
- 周常目标使玩家保持中周期推进感
- 活动任务能把版本内容转化为可执行目标

# 10.4 存在问题

- 日常完成方式相对固定，重复性高
- 长期活跃目标较少出现结构变化
- 部分任务更像“资源领取流程”，弱化了玩法差异

# 10.5 优化建议

- 增加日常任务的轻量分支结构，提高行为变化感
- 让部分任务与当期版本重点角色或机制产生联动
- 提供“自选偏好任务池”，降低任务流水线感

# 十一、长期挑战系统拆解

《崩坏：星穹铁道》的中后期留存，很大程度依赖长期挑战玩法。

# 11.1 主要玩法形态

- 模拟宇宙及其衍生模式
- 混沌回忆
- 虚构叙事
- 末日幻影
- 各类阶段性挑战内容

# 11.2 系统目标

长期挑战系统的主要职责是：

- 给中后期玩家提供明确战力验证场
- 扩大角色池与配队需求
- 延长养成系统的价值链
- 提供稳定的版本间留存抓手

# 11.3 设计优点

## 11.3.1 多模式分流

不同挑战模式对角色类型与机制偏好不同，能避免唯一解环境过早固化。

## 11.3.2 强化角色池价值

由于不同玩法需要不同功能角色，玩家会更愿意扩展队伍宽度，而不是只养一套阵容。

## 11.3.3 给“毕业”定义阶段目标

长期玩法让玩家有持续目标，不至于在主线结束后完全失去方向。

# 11.4 存在问题

## 11.4.1 环境适配压力高

当模式要求特定机制、特定输出结构、特定功能角色时，容易让玩家感觉是“卡池验证”而非纯策略挑战。

## 11.4.2 多队伍培养放大门槛

长期玩法往往不要求单一队伍强，而要求“多个角色都过线”，这对资源池压力很大。

## 11.4.3 挑战失败反馈不总是清晰

部分玩家能感知“打不过”，但不一定能明确知道：
- 是数值问题
- 是配队问题
- 是速度轴问题
- 还是机制理解问题

# 11.5 优化建议

- 增加战后分析反馈，如生存不足、出手效率不足、弱点击破覆盖不足等
- 为不同挑战模式增加更清晰的构筑指引，而不是简单推荐角色
- 缓和部分玩法对极少数环境角色的过强依赖

---

# 十二、活动系统拆解

# 12.1 活动系统的职责

《崩坏：星穹铁道》的活动系统不仅承担“发奖励”的功能，更承担：

- 版本中期内容补充
- 轻量玩法创新测试
- 角色包装和世界观延展
- 资源回补
- 活跃与回流拉动

# 12.2 活动设计优点

- 活动通常包装完整，美术与演出规格较高
- 奖励对大多数玩家有现实价值
- 活动玩法能提供短期新鲜感，缓解重复日常
- 活动与版本叙事、角色主题常有联动

# 12.3 活动系统问题

- 部分活动玩法偏短周期消费，结束后难以沉淀为长期资产
- 若活动偏叙事或偏轻量小游戏，核心战斗用户可能参与热情一般
- 活动奖励价值过高时，容易让“不参与”产生损失焦虑

# 12.4 优化建议

- 将部分高评价活动机制沉淀为常驻轻玩法或轮换玩法
- 对活动玩法进行更明确分层：剧情型、战斗型、轻休闲型
- 活动任务结构应尽量减少无意义重复刷取

# 十三、商业化系统拆解

# 13.1 核心商业化模块

| 模块 | 内容 |
|---|---|
| 角色卡池 | 新角色获取的核心入口 |
| 光锥卡池 | 提供高阶配装与商业化深度 |
| 月卡 | 提供稳定低门槛付费转化 |
| 通行证 | 提供长期活跃与付费结合 |
| 限时礼包 | 提供版本阶段性转化 |
| 资源包 | 缓解特定养成压力 |

# 13.2 商业化设计目标

- 通过角色塑造和版本叙事提升抽取欲望
- 通过角色机制与环境适配提升强度需求
- 通过养成成本和时间价值产生加速付费动机
- 通过低门槛月卡和通行证提升付费率

# 13.3 设计优点

## 13.3.1 角色价值非常强

本作商业化的核心不是纯数值售卖，而是“角色人格 + 机制差异 + 叙事包装 + 环境价值”的复合售卖。

## 13.3.2 商业化与系统循环配合紧密

玩家抽到角色后，会继续投入：
- 体力
- 材料
- 活动
- 高难挑战

这意味着一次抽卡行为会带动后续一整条系统链活跃。

## 13.3.3 低门槛付费入口清晰

月卡、通行证等设计利于轻度付费转化，降低用户首次付费心理门槛。

# 13.4 商业化系统问题

- 新角色过强或环境适配度过高时，容易引发强度焦虑
- 光锥等深度商业化模块可能放大角色完整体体验差异
- 若高难环境持续配合卡池，会引发“玩法为售卖服务”的认知风险

# 13.5 商业化结论

《崩坏：星穹铁道》的商业化成功关键在于：  
它并不是单纯卖强度，而是在卖“角色认同 + 配队价值 + 版本参与感”。  
但系统策划需要警惕的是，若环境设计过于紧贴抽卡售卖，长期会损害玩家对挑战公平性的感知。

# 十四、新手引导与前期体验

# 14.1 首次体验路径

下载进入游戏  
→ 开场剧情与角色演出  
→ 战斗教学  
→ 主线推进  
→ 初始角色获取  
→ 基础养成解锁  
→ 日常与抽卡系统逐步开放  
→ 建立首周目标

# 14.2 前期体验优点

- 制作规格高，前期吸引力强
- 回合制战斗理解门槛低
- 角色获取与剧情推进绑定，反馈密度较好
- 前期成长速度快，容易建立“在变强”的感受

# 14.3 前期体验问题

- 系统逐步开放后，信息量上升较快
- 新玩家对行迹、光锥、遗器价值的理解需要时间
- 如果抽卡结果与角色喜好不匹配，首周动力可能下降

# 14.4 优化建议

- 增加更清晰的“新角色快速成型指南”
- 为新玩家提供更透明的养成优先级建议
- 将中期常见误区前置提示，如速度、主词条、突破材料规划等

# 十五、数值与成长节奏判断

# 15.1 成长节奏分层

| 阶段 | 特征 | 系统目标 |
|---|---|---|
| 前期 | 升级快、资源反馈密、战力提升明显 | 快速建立爽感与习惯 |
| 中期 | 养成线展开、材料压力上升 | 建立长期目标与资源规划 |
| 后期 | 遗器优化、多队构筑、挑战毕业 | 延长生命周期与角色池价值 |

# 15.2 数值设计特点

- 前期通过高频升级建立强正反馈
- 中期通过突破、行迹、材料门槛拉长养成线
- 后期通过遗器随机性与高难玩法维持毕业追求

# 15.3 风险点

- 后期若只有数值追求，玩法体验容易扁平化
- 环境变动过快会导致部分角色价值波动过大
- 新玩家追赶成熟玩家需要较长时间

# 十六、成功要素总结

# 16.1 成功要素一：低门槛但不失构筑空间的战斗体系

回合制让大量用户更容易进入产品，同时配队、弱点、资源管理提供了足够的系统深度。

# 16.2 成功要素二：角色价值驱动整个系统链条

角色并不是孤立商品，而是：
- 剧情包装对象
- 战斗体验载体
- 养成资源消耗口
- 高难玩法构筑单元
- 商业化核心入口

# 16.3 成功要素三：体力与养成构成稳定日常框架

玩家每天都知道自己该做什么，这种清晰日常结构有助于提高留存稳定性。

# 16.4 成功要素四：版本供给与商业化协同成熟

新角色、新剧情、新环境、新活动构成强联动，形成稳定的版本驱动型运营模式。

# 16.5 成功要素五：中后期挑战玩法延长生命周期

多种挑战模式使角色池价值持续存在，避免产品只剩主线消费。

# 十七、主要问题诊断

# 17.1 问题一：新角色获取与可用之间存在明显断层

- 现象：玩家抽到新角色后，短期内可能缺乏足够材料让其上场
- 原因：养成线多、材料需求集中、体力限制严格
- 影响：削弱新角色抽取后的即时满足感

# 17.2 问题二：遗器随机性过强导致投入产出失衡感

- 现象：玩家长时间刷取遗器却难以获得有效提升
- 原因：主词条、副词条、强化成长多层随机叠加
- 影响：提高疲劳与挫败，削弱努力与成长的对应关系

# 17.3 问题三：中后期日常结构较稳定但变化不足

- 现象：玩家日常主要是清体力与完成固定目标
- 原因：资源副本和日常结构偏流程化
- 影响：版本空档期容易形成长草感

# 17.4 问题四：高难环境容易带来角色检测感

- 现象：部分挑战玩法对特定机制角色需求明显
- 原因：环境设计与卡池角色协同较强
- 影响：玩家容易感到挑战不是“解题”，而是“缺角色”

---

# 十八、优化方案设计

# 18.1 优化原则

- 不破坏现有商业化结构
- 不削弱角色养成长期价值
- 优先改善“获取角色后的断层感”
- 保留长线毕业目标，但降低极端挫败体验
- 提高中后期日常中的目标变化感

# 18.2 方案一：新角色快速成型支持机制

## 问题背景

新角色抽取后，如果玩家短时间内无法投入使用，会降低抽卡后的即时正反馈。

## 改动思路

- 每个版本新角色上线时，配套开放“角色成型支援任务”
- 任务围绕试玩、简单战斗、版本活动参与展开
- 奖励集中投放该角色前期突破、经验、关键行迹材料
- 目标是让玩家在较短周期内达到“可用”而非“毕业”

## 预期效果

- 缩短抽到角色到投入实战之间的时间
- 提升版本初期活跃与体验满意度
- 降低新角色带来的资源焦虑

## 风险

- 若投放过多，会压缩体力系统价值
- 若仅限新角色，可能引发老角色玩家的不平衡感

# 18.3 方案二：遗器系统增加阶段性可控纠偏

## 问题背景

遗器系统当前的随机性是长线留存核心，但极端随机会伤害投入感。

## 改动思路

- 增加“阶段性词条修正”或“限定次数重构”机制
- 玩家在长期刷取后可获得少量稀缺纠偏资源
- 仅用于提高某一阶段的可控性，而非直接毕业

## 预期效果

- 保留毕业追求
- 降低长时间零提升带来的挫败
- 增强投入与成长之间的感知关联

## 风险

- 若纠偏力度过高，会缩短遗器毕业寿命
- 需严格控制资源投放频率

# 18.4 方案三：中后期日常增加轻量变化任务池

## 问题背景

中后期日常较为固定，容易形成流程化疲劳。

## 改动思路

- 在每日目标中引入轻量变化任务池
- 例如指定属性击破、特定命途配队、特定增益条件等
- 奖励不大幅增加，重点在于改变每日行为内容

## 预期效果

- 增加日常差异感
- 提高玩家对角色池与机制的使用率
- 减少单纯“上线清体力”的机械感

## 风险

- 若变化任务太复杂，会抬高日常负担
- 需控制任务完成时间和理解成本

# 18.5 方案四：挑战玩法增加战后分析反馈

## 问题背景

玩家打不过高难内容时，不一定知道问题出在哪里。

## 改动思路

增加简洁战后分析：
- 生存不足
- 输出不足
- 弱点击破频率不足
- 行动效率不足
- 战技点循环异常

并给出轻量建议：
- 建议补充护盾/治疗
- 建议提高速度或充能
- 建议加强对群输出
- 建议补足对应属性覆盖

## 预期效果

- 提升失败后的可理解性
- 让挑战更像解题而非纯卡战力
- 对新中期玩家更友好

# 十九、对系统策划岗位的启发

通过拆解《崩坏：星穹铁道》，我认为这款产品最值得系统策划学习的点主要有以下四个：

## 19.1 系统不是孤立存在，而是围绕角色价值组织

角色获取、剧情包装、战斗表现、养成投入、挑战验证、商业化转化共同构成一个完整闭环。  
这说明系统策划设计功能时，必须从全链路而非单模块思考。

## 19.2 长线产品的关键不只是“内容多”，而是目标持续刷新

本作真正支撑长期留存的，不是单一主线，而是：
- 日常体力目标
- 周常资源目标
- 高难挑战目标
- 版本新角色与新剧情目标

系统策划要学会设计多层时间尺度上的目标结构。

## 19.3 随机性是留存工具，但必须控制挫败感

遗器系统说明：
- 随机性可以极大拉长寿命
- 但若缺少保底与纠偏，就会破坏成长正反馈

系统策划在设计长线追求时，要权衡“寿命”和“体验”。

## 19.4 成功的商业化不是脱离系统存在，而是嵌入系统链条

《崩坏：星穹铁道》的商业化不是简单卖数值，而是通过角色驱动整个系统运转。  
这对系统策划的启发是：商业化系统必须与核心体验、成长结构和版本内容高度协同。

# 二十、总结结论

《崩坏：星穹铁道》是一款非常典型且成熟的服务型二游系统样本。  
它通过低门槛回合制战斗建立广泛用户基础，通过多线角色养成和体力资源结构构建稳定日常循环，通过高难玩法和版本更新维持中长期目标，并通过角色塑造与机制包装形成强商业化转化。

从系统策划角度看，它的最大优点在于：

- 核心循环清晰
- 角色价值贯穿全系统
- 日常与版本节奏稳定
- 养成与挑战之间承接明确

其主要问题在于：

- 新角色获取后养成断层明显
- 遗器随机性过强带来的挫败累积
- 中后期日常变化不足
- 高难环境偶尔存在角色检测感

如果让我从系统策划角度优先优化，我会优先处理三件事：

1. 新角色快速成型支持
2. 遗器系统的阶段性可控纠偏
3. 高难玩法失败反馈与日常变化任务池

这三项优化不会破坏其现有商业化根基，但可以有效提升中后期玩家的成长感、理解感和持续参与意愿。

# 附录 B：面试口述版参考

## 1. 如果面试官问：你为什么选择拆解《崩坏：星穹铁道》？

我选择《崩坏：星穹铁道》，主要是因为它是一个成熟的服务型二游样本，系统结构非常完整，包含回合制战斗、角色养成、资源体力、日常任务、长期挑战、版本更新和商业化协同等模块。对系统策划岗位来说，它特别适合用来展示我对玩家行为路径、成长闭环和长期留存逻辑的理解。

## 2. 如果面试官问：你认为这款游戏最核心的系统是什么？

我认为最核心的是角色养成系统、体力资源系统和长期挑战系统。角色养成决定玩家长期投入方向，体力系统决定资源获取节奏和日常习惯，长期挑战系统则负责验证养成价值并维持中后期目标。这三个系统共同构成了它的长期循环。

## 3. 如果面试官问：你觉得它最大的系统问题是什么？

我认为最大的问题是“新角色获取与可用之间存在明显断层”。玩家抽到新角色后会有很强的即时满足，但因为等级、突破、行迹、光锥和遗器都需要投入，短期内未必能真正上场，这会削弱抽卡之后的正反馈。我认为这是一个典型的系统衔接问题，而不是单点数值问题。

## 4. 如果面试官问：你最想优化哪部分？

我会优先优化三个部分。第一是新角色快速成型支持，让玩家在版本初期能更快体验新角色。第二是遗器系统加入阶段性纠偏机制，降低极端随机性带来的疲劳。第三是给高难玩法增加战后分析反馈，让玩家知道自己输在构筑、数值还是机制理解上。

# 附录 C：面试追问与回答要点

## Q1：遗器随机性如果降低，会不会缩短游戏寿命？

会有这个风险，所以我的思路不是大幅降低随机性，而是增加“阶段性纠偏”。也就是保留长期刷取与毕业追求，但减少长时间没有有效提升的极端挫败体验。重点是改善体验下限，而不是直接提高毕业效率上限。

## Q2：新角色快速成型支持会不会影响商业化和体力价值？

如果直接大量发资源，会有影响。所以我更倾向于做“限时定向支持”，目标是让角色达到可用状态，而不是直接毕业。这样既能保留体力系统价值，也能减少抽到角色却无法体验的断层。

## Q3：这款游戏为什么要做这么多高难玩法模式？

我认为核心原因是单一模式无法长期支撑多角色池价值。通过不同挑战模式偏好不同角色和机制，可以提高角色池宽度价值，延长养成系统寿命，也让版本环境有更多变化空间。

## Q4：你觉得这款游戏更依赖内容驱动还是数值驱动？

我认为它本质上是“内容驱动外壳 + 数值成长中轴 + 角色商业化核心”的混合结构。剧情和版本负责拉动关注，养成和挑战负责支撑长期游玩，角色投放负责把关注转化为持续参与和付费。

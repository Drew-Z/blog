# 一、《崩坏：星穹铁道》面试口述稿

## 1. 30 秒版本

我选择拆解《崩坏：星穹铁道》，主要是因为它是一个很典型的长线养成型产品，系统结构比较完整，适合从系统策划角度分析。我重点拆了它的战斗、养成、体力资源和版本驱动逻辑。我的核心判断是，这款游戏的优势不在于战斗特别深，而在于它用低门槛回合制，把角色获取、资源消耗、角色成长和版本更新稳定串成了一个长期循环。它的主要问题则集中在后期日常趋平、新角色成型慢，以及遗器系统的随机挫败感。

## 2. 1 分钟版本

我在拆解《崩坏：星穹铁道》时，重点关注的是它如何把“抽角色”变成“长期养角色”的留存闭环。  
我认为这款产品的核心结构是：

战斗获取资源  
→ 投入角色、光锥、行迹、遗器等多线养成  
→ 角色逐步成型  
→ 去验证高难内容  
→ 再被新版本角色和活动拉回下一轮循环

它做得比较好的地方有三点。  
第一，核心循环非常清晰，玩家通常知道自己下一步该做什么。  
第二，回合制和自动战斗降低了门槛，比较适合长线陪伴型产品。  
第三，多线养成虽然复杂，但优先级相对明确，玩家不会一上来就完全迷失。

我认为它最大的几个问题是：  
一是版本中段容易退化为清体力，目标波动不够；  
二是新角色从抽到到真正能上场之间有明显养成门槛；  
三是遗器系统多重随机叠加，后期投入和回报不稳定。

所以我的优化思路主要是三个方向：  
一是做新角色快速可用化机制；  
二是给遗器系统增加更稳定的纠偏手段；  
三是在高难和纯日常之间增加中强度常驻目标层。

## 3. 3 分钟版本

我选择《崩坏：星穹铁道》作为作品，一方面是因为它是很典型的二游长线养成产品，系统结构完整；另一方面是因为它很适合从系统策划角度拆“玩法、养成、资源、版本、商业化”之间的联动关系。

我拆这款游戏时，首先不是把注意力放在剧情和角色人气上，而是看它的长期循环怎么成立。  
我的核心判断是，它的战斗系统其实承担的是一个“低门槛入口和养成验证器”的角色，而真正驱动中长期留存的，是养成系统和版本角色投放。

如果用一句话概括它的闭环，就是：

低门槛回合制战斗  
→ 获取资源  
→ 投入角色、光锥、行迹、遗器养成  
→ 角色逐步成型  
→ 打更高难内容  
→ 围绕新角色和新版本重新规划资源  
→ 回到下一轮养成循环

我觉得它做得最好的地方有几个。  
第一，玩家目标很清晰。前中期通常知道自己缺什么、该刷什么。  
第二，回合制和自动战斗让它的日常负担相对可控，这对长线产品非常重要。  
第三，多线养成虽然看上去复杂，但它通过阶段性目标把复杂度分散了。比如前期主要关注等级、行迹和基础装备，后期才逐渐进入遗器精调，所以玩家不会一开始就直面最痛苦的随机优化。

但它的问题也很典型。  
第一个问题是中后期日常趋平。主线和活动内容消耗完之后，很多玩家的行为会收缩成“上线清体力”，这说明它的版本中段波动不足。  
第二个问题是新角色成型门槛高。玩家抽到角色时情绪峰值很高，但接下来往往会被材料、光锥、遗器等多条成长线拖慢，导致“抽到的快乐”和“能用的快乐”没有很好衔接。  
第三个问题是遗器系统随机性过强。它本来承担的是后期长线追求，但因为随机维度太多，容易让玩家产生高投入低反馈的疲劳感。

所以如果从系统策划角度优化，我会优先做三件事。  
第一，做新角色快速可用化机制，不是直接送毕业，而是缩短抽到到可用的距离，比如定向基础材料、过渡套装、推荐养成导航。  
第二，给遗器系统加入更明确的纠偏机制，比如更稳定的定向获取或有限修正，降低纯随机挫败。  
第三，在高难和纯日常之间补一层中强度常驻目标，让玩家在版本中段仍然有“今天上线不只是清体力”的理由。

这份拆解我主要想体现的是：  
我不仅能分析养成系统本身，还能把体力节奏、版本角色投放、高难验证和商业化联动放到同一套长期结构里去看。

# 二、《明日方舟：终末地》面试口述稿

## 1. 30 秒版本

我选择拆解《明日方舟：终末地》，主要是因为它不像传统二游那样只有战斗和养成两层，而是明显存在战斗、探索、采集、基地和制造等多系统耦合。我的核心判断是，它真正值得分析的不是单一玩法，而是这些系统能不能形成一个长期运转的闭环。它的优势在于多系统联动潜力很强，但主要风险也很明显，就是系统门槛高、劳动感重、以及战斗可能被工具化。

## 2. 1 分钟版本

我拆《明日方舟：终末地》时，最关注的是它是不是能从“有很多系统”变成“这些系统彼此支撑”。  
我认为它潜在的核心闭环是：

探索和战斗  
→ 获取资源  
→ 投入基地建设和制造  
→ 提高效率和系统能力  
→ 解锁更复杂区域和目标  
→ 再回到探索、采集和推进

如果这个闭环成立，它的长期价值就不只是角色养成，而是“系统扩张”和“效率成长”。

我觉得它最值得关注的优点有三点。  
第一，多系统联动潜力很强，不是单点玩法叠加。  
第二，基地和制造系统有机会成为真正的中层驱动，而不是装饰型家园。  
第三，探索、采集和建设发生在空间中，参与感会比纯菜单循环更强。

它最大的风险也很明确。  
一是系统太多，前期理解成本容易失控；  
二是采集、搬运、制造如果太手动，会让玩家觉得在打工；  
三是如果后期都在看基地，战斗可能退化成资源获取工具。

所以我的优化思路是：  
前期先压缩成最小可运行循环；  
中后期逐步开放自动化和效率工具；  
同时强化战斗和建设的互相回馈，避免系统割裂。

## 3. 3 分钟版本

我选择《明日方舟：终末地》做作品，主要是因为它非常适合展示系统策划对“多系统联动”的理解。  
相比已经成熟的长线产品，它更像一个结构型项目：战斗、探索、采集、基地、制造这些系统看起来都不只是附属，而是有可能共同构成长期驱动。

我在拆这款游戏时，刻意把分析分成两层。  
第一层是已公开和可观察的内容，比如即时战斗、3D探索、资源采集、基地设施和制造感。  
第二层是基于这些内容做系统策划推演，比如它的长期闭环能不能跑通、基地系统到底是不是中层驱动、后期留存会更依赖内容更新还是系统扩张。

我的核心判断是，《终末地》真正的价值不在于“3D 化”或者“动作化”，而在于它是否能让这条链路成立：

探索和战斗  
→ 资源采集  
→ 基地建设和制造  
→ 效率提升  
→ 角色和系统共同成长  
→ 解锁更复杂区域和目标  
→ 再回到下一轮推进

如果这条链路成立，它就不是普通的角色养成产品，而是一个更偏“结构成长”的长线产品。玩家的长期满足感不只是数值变高，而是看到自己建立的系统越来越能运转。

我认为它最大的优势有三个。  
第一，多系统之间存在天然闭环潜力。战斗可以提供推进和资源，探索提供空间路径，基地和制造承接中长期目标。  
第二，基地和制造如果做成真正的效率系统，会形成非常强的中层驱动。玩家不是只在练角色，而是在经营一个越来越高效的结构。  
第三，空间化体验会增强参与感。玩家不是在菜单里点合成，而是在场景中感知采集、建设和扩张，这种沉浸对长期产品很有价值。

但它的问题也非常典型。  
第一，系统复杂度很容易失控。战斗、探索、采集、建设、制造如果同时压到前期，玩家会迅速迷失。  
第二，劳动感风险很高。工业和基地系统天然容易变成重复劳动，如果中后期没有自动化演进，玩家会觉得自己在完成琐事。  
第三，战斗系统存在被工具化的风险。如果后期所有注意力都被基地效率吸走，战斗就可能只剩“拿材料的门槛”，这会削弱角色价值和表层乐趣。  
第四，多系统共用资源时，容易放大焦虑。玩家会不断在“先建基地还是先练角色”之间疲于取舍。

所以我的优化建议主要围绕四个方向。  
第一，前期严格控制复杂度，建立最小可运行循环。比如先只开放探索、战斗、基础采集和一个核心设施，让玩家先理解为什么建设有意义。  
第二，中后期逐步开放自动化和效率工具，让玩家从“手搓流程”进入“优化结构”的阶段。  
第三，强化战斗与建设的回馈关系，让基地成长不仅是资源效率，也能反馈战斗准备或功能支持。  
第四，用阶段性目标树替代平铺任务，让玩家始终知道“当前最重要的一件事”。

这份作品我主要想体现的是：  
我能在信息不完全成熟的项目里，区分事实和推演，并从系统联动、复杂度控制、劳动感风险和长期结构角度去分析一个复合型产品。

# 三、《王者荣耀国际服》面试口述稿

## 1. 30 秒版本

我拆解《王者荣耀国际服》，重点不是分析 MOBA 局内战斗本身，而是看它如何通过外围系统把高情绪波动的竞技体验转化成长期留存。我重点分析了新手承接、英雄池、排位赛季、活动、社交和商业化。我的核心判断是，这类产品的长期价值不只来自“打一局很好玩”，而来自“赛季目标、社交关系和身份表达”能不能持续承接玩家。

## 2. 1 分钟版本

我认为《王者荣耀国际服》的系统设计重点，在于核心对局之外的外围结构。  
它的基本闭环可以概括成：

打一局对局  
→ 获得胜负反馈、段位进度、活动进度和社交反馈  
→ 推进赛季目标、扩展英雄池或强化身份表达  
→ 再进入下一局

它做得比较好的地方有四点。  
第一，MOBA 对局本身复玩性很强。  
第二，排位和赛季结构很成熟，能持续提供长期目标。  
第三，社交和开黑和核心玩法天然契合，留存价值很高。  
第四，商业化更多围绕皮肤和身份表达，和竞技公平相对兼容。

它的主要问题也很典型。  
一是竞技挫败容易连续放大，比如连败和掉段；  
二是新手从“会玩”到“长期玩”之间有断层；  
三是轻度用户在赛季体系中的目标感偏弱；  
四是活动任务如果设计不好，容易反过来干扰正常对局。

所以我的优化方向是：  
给新手补英雄池成长路线；  
给轻度用户设计低压赛季目标层；  
让活动任务和正常对局行为更一致；  
同时增强失败后的过程反馈，缓冲纯输赢评价。

## 3. 3 分钟版本

我选择《王者荣耀国际服》作为第三份作品，是希望补足自己对竞技服务型产品的理解。  
因为和《崩坏：星穹铁道》这种养成产品不同，MOBA 的核心玩法天然具有高复玩性，所以它的系统策划重点，不是用资源养成去驱动玩家，而是用外围系统去承接竞技情绪、包装长期目标。

我在拆这款产品时，故意把分析分成两层：  
第一层是核心对局，也就是 5v5 实时对抗、推塔、资源争夺和团队配合。  
第二层是对局外系统，包括新手承接、英雄池、排位、赛季、活动、社交和商业化。  
我的核心判断是：  
对这类产品来说，局内玩法决定“这局值不值得玩”，而局外系统决定“为什么一个赛季、多个赛季持续玩下去”。

如果概括它的长期结构，可以理解为：

对局胜负  
→ 段位变化 / 活动进度 / 社交反馈  
→ 英雄池与身份积累  
→ 赛季目标推进  
→ 回到下一轮对局

我觉得它做得最好的地方有几个。  
第一，MOBA 对局本身天然具备高重复游玩价值。不同英雄、队友、版本和局势让每局都不一样。  
第二，排位和赛季系统很成熟，它把无限重复的对局切成了一个个阶段性的征程，让玩家一直有“这一赛季还要再冲一点”的目标。  
第三，社交系统与核心玩法高度适配。开黑、好友、组队语音天然能提高留存，甚至很多时候“朋友在线所以我上线”比奖励更强。  
第四，商业化和竞技公平相对兼容。主要是皮肤、战令、展示和身份表达，而不是直接卖对局强度。

但这类产品的问题也非常典型。  
第一，竞技挫败容易被连续放大。连败、掉星、队友问题会迅速伤害情绪，如果外围系统不给缓冲，玩家很容易直接下线。  
第二，新手从“会放技能”到“知道打哪条路、该练哪些英雄、为什么打排位”之间有明显断层。MOBA 真正的门槛不只是操作，而是中层理解。  
第三，轻度用户在赛季系统中的覆盖不够充分。如果玩家不热衷冲分，他很可能只剩“偶尔打一两局”的动力。  
第四，活动任务如果设计不当，会干扰正常竞技体验。比如要求奇怪行为或强迫玩不熟悉的英雄，会影响自己和队友的体验。

所以我的优化建议主要有四个方向。  
第一，给新手到中期补一个英雄池成长路线，不只是送英雄，而是告诉玩家：当前适合哪条路、哪些低门槛英雄值得练、怎么建立基础池子。  
第二，为轻度用户设计低压赛季目标层，让不以冲高段位为核心的玩家也能感受到赛季参与价值。  
第三，重构活动任务，让它尽量和正常对局行为一致，比如参团、推塔、协助，而不是反玩法目标。  
第四，强化失败后的过程反馈，让玩家在输了以后不只是看到“失败”，还能知道自己做对了什么、下一步该改什么。

这份作品我主要想体现的是：  
我能把竞技产品拆成“核心对局”和“外围承接”两层，并从赛季、活动、社交和身份表达这些系统去理解长期留存，而不是只停留在英雄强度和平衡层面。

# 四、三篇作品的统一收束话术

## 1. 如果面试官问：这三篇作品你想共同体现什么？

这三篇作品分别代表养成驱动型、复合系统型和竞技服务型产品。  
我想共同体现的是：虽然赛道不同，但我分析的核心问题是一样的——一款产品如何把单次体验转化成长期留存。我会重点找出每款产品真正的中层驱动系统，再去分析它和资源、目标、留存、商业化之间的关系。

## 2. 如果面试官问：你最看重哪项系统策划能力？

我最看重的是“把玩家感受拆成结构”的能力。  
也就是能判断一个系统为什么存在、它驱动了什么行为、和其他系统怎么联动、问题的根因是什么，以及优化后可能带来什么副作用。

## 3. 如果面试官问：三款游戏里你最想参与哪类项目？

如果从兴趣上说，我会对像《明日方舟：终末地》这种多系统联动型项目非常感兴趣，因为它更考验系统结构设计能力；  
但从上手和方法论积累角度，我觉得《崩坏：星穹铁道》这种长线养成产品也非常适合训练系统策划的基本功；  
而《王者荣耀国际服》则让我更理解竞技产品中外围系统的重要性。
